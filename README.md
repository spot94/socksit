# SocksIt

Прозрачный проксификатор отдельных приложений под Windows 10/11 (x64): заворачивает
трафик выбранных программ в SOCKS5-прокси автоматически — приложения запускаются как
обычно, ничего не нужно стартовать через отдельный лаунчер.

## Что это и как работает

Служба (LocalSystem) поднимает виртуальный сетевой адаптер (TUN, Wintun) и запускает
движок **sing-box** как дочерний процесс. Приложения выбираются по имени или полному пути
процесса (например `chrome.exe`, `claude.exe`):

- их трафик перехватывается через TUN;
- DNS-запросы этих приложений подменяются на «fake-ip», чтобы соединения гарантированно
  шли через туннель;
- по правилам маршрутизации трафик выбранных приложений уходит в **SOCKS5-прокси**
  (TCP и UDP), остальной трафик идёт **напрямую**;
- при обрыве туннеля **kill-switch** не даёт проксируемым приложениям «утечь» напрямую.

Матчинг идёт и по имени, и по пути процесса, поэтому ловятся даже дочерние/песочные
процессы (например вкладки и служебные процессы Electron-приложений).

Управление — окно панели и значок в трее (работают в сессии пользователя) — общается со
службой по защищённому локальному каналу. Пароль к прокси хранится в зашифрованном виде
(DPAPI) и не пишется в конфиг.

Датаплейн — официальный **sing-box** (распространяется отдельным бинарём под GPLv3).

## Установка и запуск

Требуется: **Windows 10/11 x64**, права **администратора** для установки службы и
**Microsoft Edge WebView2 Runtime** (на Win10/11 обычно уже есть; иначе — поставить
«Evergreen WebView2 Runtime» от Microsoft).

1. Достаточно одного `socksit.exe`. Движок `sing-box.exe` можно положить рядом, но если его
   нет — он **скачается автоматически** при установке (из официального репозитория sing-box,
   с проверкой по контрольной сумме; при недоступности — из релизов SocksIt).
2. Запустите `socksit.exe` — откроется панель. Нажмите **Set up** (установит службу, при
   необходимости скачает движок и запустит). Без прав администратора кнопки установки
   неактивны — нажмите **Restart as administrator**.
3. В разделе **Settings** укажите адрес и порт SOCKS5-прокси (при необходимости логин/пароль)
   и добавьте приложения. Для веб-приложений (сайты в браузере) добавляйте **браузер**
   (`chrome.exe`, `msedge.exe`). Нажмите **Save & apply**.
4. Убедитесь, что трафик реально идёт через прокси, в разделе **Statistics** (столбец
   **Via** = `proxy`).

Удаление — кнопка **Uninstall** в панели или `socksit uninstall` от администратора.

Конфигурация и логи хранятся в `%ProgramData%\SocksIt\` (`socksit.yaml`, `socksit.log`).

## Использование — панель (GUI)

Запуск: двойной клик по `socksit.exe` (или `socksit`). Единое окно, слева — меню разделов.
Внизу слева — переключатель **темы** (светлая/тёмная/системная) и **языка** (EN/RU).

- **Dashboard** — состояние службы и движка, прокси и число приложений. Переключатель
  **Proxying** ставит проксирование на паузу / возобновляет без остановки службы. Кнопки
  **Set up / Start / Stop / Uninstall** и **Test proxy** (проверка доступности прокси).
- **Settings** — адрес/порт/логин/пароль прокси; режим **allowlist** (проксируются только
  выбранные) или **blocklist** (все, кроме выбранных); **kill-switch**; **UDP**; **Show in
  tray**; «прямые» подсети (в обход прокси); список приложений (**Add**, **Browse…** — выбрать
  .exe из проводника, **Remove**; поле с автодополнением из запущенных процессов).
  **Save & apply** применяет на лету (если служба остановлена — сохранит в файл и применит
  при старте). Блок **Config source** (опц.) — «управляемый конфиг»: если задать URL, конфиг
  тянется оттуда и применяется автоматически (при старте и по интервалу), локальные правки
  перезаписываются. По умолчанию требуется подпись (`<url>.sig`, вшитый ключ) — можно отключить
  для доверенной сети. Кнопка **Fetch now** обновляет сразу.
- **Statistics** — активные соединения: **Процесс · Назначение · Сеть · Через · Отпр. ·
  Получ.** Клик по заголовку сортирует столбец, есть фильтр и авто-обновление. `Через =
  proxy` означает, что соединение реально проксируется.
- **Logs** — журнал службы и движка; выбор числа строк и авто-обновление.
- **Diagnostics** — отчёт «почему приложение не проксируется»: проверяет службу, движок,
  конфиг, доступность прокси и какие из настроенных приложений сейчас запущены.
- **About** — версия, версия движка, путь к конфигу и блок **Updates** (источник обновлений,
  канал, режим off/notify/auto, интервал, прокси для проверок, «Проверить сейчас» и «Установить
  обновление» — скачивание с проверкой контрольной суммы и перезапуск службы с откатом при неудаче).

**Трей** (значок в области уведомлений): строка статуса, чекбокс **Proxying enabled**
(пауза/возобновление), **Open SocksIt** (открыть панель) и версия. Значок присутствует,
пока служба установлена (и включён параметр «Show in tray»); отдельного пункта «Выход» нет.

## Использование — командная строка (CLI)

| Команда | Что делает |
|---|---|
| `socksit` | Открыть панель управления (без аргументов). |
| `socksit gui` | Открыть панель сразу на разделе Settings. |
| `socksit tray` | Значок в трее. |
| `socksit setup` | Установить службу, применить пресет (если есть), запустить. Нужен администратор. |
| `socksit install` \| `socksit uninstall` | Зарегистрировать / удалить службу Windows. Администратор. |
| `socksit service` | Запуск под управлением Windows (SCM). |
| `socksit run` | Интерактивный запуск (для разработки). |
| `socksit gen -c socksit.yaml [-o config.json]` | Сгенерировать конфиг движка из `socksit.yaml`. |
| `socksit check -c socksit.yaml [-engine sing-box.exe]` | Проверить сгенерированный конфиг (`sing-box check`). |
| `socksit version` | Версия приложения и движка. |

Пример настройки без GUI — отредактировать `%ProgramData%\SocksIt\socksit.yaml`:

```yaml
proxy:
  address: 10.0.0.1
  port: 1080
apps:
  - chrome.exe
  - claude.exe
mode: allowlist        # allowlist | blocklist
kill_switch: true
```

Изменения в файле применяются автоматически (служба следит за конфигом). Логин/пароль в
YAML не пишутся — их задаёт панель (шифруются службой).
