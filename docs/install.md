# Установка и развёртывание (DOC-3)

## Требования

- Windows 10/11 x64.
- Права администратора (для установки сервиса и создания TUN-адаптера).
- SOCKS5-прокси (адрес и порт; логин/пароль опционально).

## Вариант A — подписанный инсталлятор (целевой; U11, ещё не собран)

Планируемый путь для конечных пользователей: MSI (WiX), который регистрирует сервис
`SocksIt` (LocalSystem, автостарт), кладёт движок и `wintun.dll`, ставит автозапуск трея и
создаёт дефолтный конфиг. Сборка инсталлятора требует WiX и (для отсутствия
SmartScreen-трения) сертификата подписи; без сертификата допускается неподписанная сборка.
Статус — см. `BUILD-PROGRESS.md`.

## Вариант B — сборка из исходников (текущий, local-first)

Нужен Go 1.25+ (для UI-частей — C-компилятор для cgo; ядро собирается с `CGO_ENABLED=0`).

```powershell
# сборка бинарника ядра
$env:CGO_ENABLED = "0"
go build -o bin\socksit.exe .\cmd\socksit

# движок sing-box лежит в assets\bin\sing-box.exe (pinned v1.13.14)
```

Регистрация сервиса (из-под администратора):

```powershell
bin\socksit.exe install     # создаёт службу SocksIt (LocalSystem, автозапуск), запускает `socksit service`
bin\socksit.exe uninstall   # останавливает и удаляет службу
```

Для разработки без службы:

```powershell
bin\socksit.exe run
```

`run` запускает тот же Runtime в консоли. **Важно:** Wintun-адаптер создаётся только из-под
учётной записи **SYSTEM**, поэтому `run` даже с правами администратора не поднимет туннель —
он остановится на создании адаптера (и «повиснет», ожидая валидный конфиг, если прокси не задан).
Реальное проксирование запускается **через установленную службу** (`install`, работает как
LocalSystem). `run` полезен для проверки генерации конфига и логики; для проверки самого прокси
без TUN используйте loopback-тест из [testing-now.md](testing-now.md).

## First-run

При первом запуске сервис создаёт `%ProgramData%\SocksIt\socksit.yaml` с дефолтами:
пустой `allowlist` (пока приложения не добавлены — ничего не проксируется), `kill_switch: true`.
Отредактируйте его (см. [`socksit.example.yaml`](socksit.example.yaml) и
[usage.md](usage.md)) и укажите адрес прокси. Изменения применяются на лету (hot-reload).

Проверить корректность конфигурации без запуска службы:

```powershell
bin\socksit.exe check -c %ProgramData%\SocksIt\socksit.yaml
```
