# Руководство по использованию (DOC-4)

Настраивать SocksIt проще всего через **окно настроек**: значок в трее → «Open app list…»
(или `socksit gui`). Там задаются прокси, логин/пароль, режимы, kill-switch и список приложений —
**без ручной правки файла**. Кто предпочитает файл — те же параметры лежат в
`%ProgramData%\SocksIt\socksit.yaml` (путь печатается при старте службы). Любые изменения
применяются автоматически (hot-reload): служба перегенерирует конфиг движка и мягко перезапускает его.

## Быстрый старт

1. Укажите прокси и приложения в `socksit.yaml`:

```yaml
proxy:
  address: 192.0.2.10   # ваш SOCKS5-сервер
  port: 1080
  username: ""          # пусто = без авторизации
  password: ""
  udp: true

apps:
  - chrome.exe          # по имени процесса
  - Telegram.exe
  # - C:\Games\game.exe # или по полному пути

mode: allowlist         # allowlist = проксируются только эти; blocklist = все, кроме этих
kill_switch: true       # true = при падении туннеля проксируемые приложения блокируются (без утечки IP)
```

2. Сохраните файл — служба применит изменения сама. Запускайте приложения как обычно;
   трафик из списка пойдёт через прокси, остальное — напрямую.

## Режимы

- **allowlist (по умолчанию)** — проксируются только перечисленные приложения.
- **blocklist** — проксируется всё, кроме перечисленных.
- **захват трафика** — режим один (полный захват): весь трафик заходит в туннель, а маршрутизация
  по процессу решает proxy/direct. Следствие — SocksIt не уживается с полнотуннельным VPN.
- **kill_switch** — `true` держит проксируемые приложения без интернета, пока туннель не
  поднимется (защита от утечки реального IP); `false` (fail-open) пускает их напрямую.

## CLI

```powershell
socksit gen   -c socksit.yaml [-o config.json]   # показать/сохранить сгенерированный конфиг движка
socksit check -c socksit.yaml                     # проверить конфиг через `sing-box check`
socksit install | uninstall                       # регистрация/удаление службы (admin)
socksit service | run                             # запуск как службы / интерактивно
socksit version
```

## Диагностика

- Аудит действий: `%ProgramData%\SocksIt\audit.log` (кто → что → над чем).
- Статистика по приложениям (соединения/трафик) — через окно статистики (U10) поверх
  Clash API движка на loopback; до появления UI доступна как JSON через IPC-операцию `stats`.
- Проверка, что прокси действительно меняет IP: сравните внешний IP до/после в проксируемом
  приложении (например, страницей проверки IP) — как в приёмочных примерах AE1/AE2 плана.
