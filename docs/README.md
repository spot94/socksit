# SocksIt — описание проекта (DOC-1)

## Кратко

SocksIt — локальный однопользовательский Windows-сервис (Win10/11 x64), который
**прозрачно** заворачивает трафик выбранных приложений в SOCKS5-прокси (TCP+UDP+DNS)
по имени/пути процесса, **без** ручного запуска приложений через какой-либо GUI.
Задача — надёжнее ProxiFyre (без «отвалов» перехвата) и автоматичнее SocksCap64.

## Архитектура

Датаплейн — официальный **sing-box** (pinned v1.13.14), запускаемый как
**поднадзорный дочерний процесс** и управляемый сгенерированным `config.json`.
Поскольку Wintun-адаптер открывается только из-под SYSTEM, продукт разделён на
**сервис LocalSystem** (вся привилегированная сетевая работа) и **приложение
пользовательской сессии** (трей + GUI), общающиеся по защищённому named pipe.

```
Пользовательская сессия                Session 0 (LocalSystem, socksit.exe service)
┌───────────────────────┐    pipe     ┌───────────────────────────────────────────┐
│ панель (WebView2)      │◄──go-winio─►│ ipc-сервер (SDDL deny-by-default)          │
│ + трей (systray)       │  SDDL ACL   │ Runtime: config→generate→check→supervise   │
└───────────────────────┘             │ supervisor (Job Object kill-on-close)      │
                                       │ netmon (winipcfg) → self-heal · netstate   │
                                       │ secret (DPAPI)  · audit-log                │
                                       └───────────────┬────────────────────────────┘
                                                       ▼ spawn + config.json
                                          sing-box (Wintun TUN, stack=system) → SOCKS5
```
Панель — единое окно на **WebView2** (чистый Go, без cgo), со светлой/тёмной темой и языками
EN/RU; трей минимальный и жёстко привязан к службе.

Ключевые решения и обоснования — в
[плане](plans/2026-07-13-001-feat-socksit-per-app-socks5-plan.md) (Key Technical Decisions):
subprocess из-за GPLv3, `stack: system` (не gVisor), единый greedy-захват (`auto_route`),
матчинг по имени и пути процесса, kill-switch через fake-ip, разделение привилегий,
Job-Object teardown, DPAPI, self-heal через `auto_detect_interface`.

## Потоки данных и обрабатываемые данные

SocksIt — сетевое ПО и **не передаёт никакие данные в LLM/модели** (см. DATA-1/DATA-2 в
[`requirements-baseline.yaml`](../requirements-baseline.yaml), статус N/A). Обрабатываемые
данные: (1) конфигурация пользователя (список приложений, адрес прокси); (2) креды SOCKS5 —
хранятся зашифрованно DPAPI, не в git; (3) сетевой трафик выбранных приложений — проксируется,
но не логируется по содержимому. Аудит-лог фиксирует только действия пользователя над конфигом.

## Состояние сборки

Реализованы: ядро (генерация конфига проходит `sing-box check`, супервайзор, DPAPI-секреты,
IPC+аудит, сервис-хост), **панель WebView2** (Dashboard/Settings/Statistics/Logs/Diagnostics/
About, темы, EN/RU), **трей** (привязан к службе), **канал обновлений** через GitHub с
Ed25519-подписью (Этап 1 — проверка+уведомление). Впереди: применение обновлений (Этап 2),
автозагрузка движка при установке, MSI-инсталлятор, Authenticode, живая приёмка AE1–AE8
(нужны admin + интерактивная Windows). Актуальный статус: [`BUILD-PROGRESS.md`](BUILD-PROGRESS.md).
